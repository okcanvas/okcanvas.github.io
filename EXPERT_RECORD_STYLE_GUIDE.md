# Expert Record Style Guide

이 포트폴리오의 기준은 “시니어처럼 보이는 문장”이 아니라 “전문가가 남긴 판단 기록”입니다.

## 유지할 문장 구조

- 문제를 기능 요구가 아니라 운영에서 깨지는 지점으로 정의한다.
- 선택한 기술보다 선택 기준과 버린 선택을 기록한다.
- API, worker, gateway, DB, cache, media server, Agent 실행 계층의 책임 경계를 분리한다.
- 정상 동작보다 실패, 재처리, rollback, audit, trace 가능성을 먼저 검증한다.
- 성과는 과장하지 않고 운영 흐름이 어떻게 바뀌었는지로 표현한다.

## 피해야 할 문장

- “좋다”, “나쁘다”, “어려웠다”, “고생했다”처럼 감상에 가까운 표현
- “처음에는”, “나중에는”, “느꼈다”처럼 회고문 중심의 흐름
- “강력하다”, “충분하다”처럼 근거 없는 형용사
- “할 수 있었다”가 반복되는 수동적 표현
- 특정 도구의 우열을 단정하는 표현

## 권장 표현

- “이 선택의 기준은 …였다.”
- “이 계층의 책임은 …로 제한했다.”
- “운영 검증 기준은 …였다.”
- “이 구조는 …을 설명하기 위한 것이다.”
- “실패 시 …로 되돌릴 수 있어야 했다.”
- “이 프로젝트의 결론은 …이다.”
