# STEP027 Expert Record Review

## 수정 기준

이번 단계에서는 첫 화면에서 직급과 규모를 앞세우는 표현을 제거했습니다. 포트폴리오의 인상은 직급명이 아니라, 문제 정의·책임 경계·운영 검증·장애 대응 기준에서 드러나야 합니다.

## 주요 변경

- 홈 타이틀을 직급 강조형 표현에서 `Backend Platform Engineering Record`로 변경했습니다.
- 첫 문단에서 구체 규모를 앞세우는 표현을 제거했습니다.
- 경험 배경과 Kafka 장애 모니터링 문서의 규모 표현을 `다중 고객사 ERP 환경`으로 낮췄습니다.
- config metadata와 sidebar 문구에서 직급 강조 표현을 제거했습니다.
- 기존 STEP026 리뷰 문서는 삭제하고 STEP027 리뷰 문서로 대체했습니다.

## 문장 기준

좋은 전문가 기록은 직급이나 숫자를 앞세우지 않습니다. 대신 다음 항목이 문장 안에 남아야 합니다.

- 어떤 운영 문제를 다뤘는가.
- 어떤 계층의 책임을 분리했는가.
- 어떤 선택지를 버렸는가.
- 실패·감사·rollback·추적 가능성을 어떻게 검증했는가.
- 결과적으로 운영 흐름이 어떻게 달라졌는가.

## 검토 결과

- 첫 화면 직급 강조 표현 제거 완료
- 첫 화면 규모 강조 표현 제거 완료
- 직급 강조 표현 제거 완료
- 구체 규모 강조 표현 제거 완료
- 문서 톤은 전문가의 기술 기록 기준으로 유지
